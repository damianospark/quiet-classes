4강은 "협업과 전략 기획"을 완성하기 위한 프로젝트 기반 수업입니다. 좌측 사이드바 단계에 따라 이동하며 각 실습을 순차적으로 진행하세요.

역할 정리

  • A (전략 담당자): 기획 아이데이션·KPI·Slides 총괄
  • B (콘텐츠 담당자): 자기소개서 작성, 문장 교정, 공유 메시지 관리
  • 팀장: Notion 일정·산출물 통합, 회고 수집 및 보고

실습 운영 방식: 실제 팀이 없는 상황을 가정해 한 명이 여러 역할을 수행합니다. 기본 Chrome 창은 팀장 시점으로, 시크릿 창 1은 A 담당자, 시크릿 창 2는 B 담당자처럼 창마다 역할을 고정해 작업하세요. 각 창에서 생성한 Google Workspace 문서는 공유 버튼 → 링크 복사로 서로 전달하며, Notion 페이지에 링크를 모아 교차 확인합니다.

프로젝트 워크플로

한 명이 세 개의 창을 사용해 역할을 분담합니다. 기본 Chrome 창은 팀장 역할, 시크릿 창 1은 A 담당자, 시크릿 창 2는 B 담당자 전용으로 두고 각 창에서 문서를 작성합니다.

준비 체크리스트

  • [ ] 기본 Chrome 창에서 Notion 빈 페이지를 만들고 공유 링크를 복사해 둔다.
  • [ ] 시크릿 창 1(A)에서 Google Docs 기획 문서를 생성하고 공유 버튼 → 링크 복사 후 Notion에 기록한다.
  • [ ] 시크릿 창 1(A)에서 Google Sheets KPI 시트를 생성하고 공유 링크를 복사한다.
  • [ ] 시크릿 창 2(B)에서 Google Docs 자기소개서 문서를 생성하고 공유 링크를 복사한다.
  • [ ] 시크릿 창 2(B)에서 Google Slides 전략 요약 슬라이드를 생성하고 공유 링크를 복사한다.
  • [ ] Notion 페이지 상단에 문서 링크 모음 테이블을 만들고 역할·문서 유형·공유 링크를 정리한다.

브라우저 운영 팁

  • 시크릿 창은 Shift + Command + N(macOS) 또는 Ctrl + Shift + N(Windows)로 연다.
  • 창 제목에 역할명을 붙여 헷갈리지 않도록 한다.
  • 모든 문서는 링크 소지자 보기/댓글 허용으로 설정해 다른 창에서도 접근 가능하게 만든다.

성과물

역할별 문서 링크를 정리한 Notion 페이지 스크린샷과 공유 URL을 확보한다.

<보완요청> 실습 환경 브라우저 구성 캡처 삽입

프로젝트를 한눈에 관리할 수 있도록 Notion 테이블 보드를 만든다. 타임라인 대신 행마다 마일스톤을 배치해 역할과 상태를 체계적으로 추적한다.

  1. Notion에서 새로운 데이터베이스(테이블 뷰)를 생성하고 이름을 프로젝트 보드로 지정한다.
  2. 기본 이름 열은 마일스톤 제목(예: 아이데이션, KPI 설계, Slides 제작, 자기소개서, 통합, 품질검토)을 입력한다.
  3. 다음 속성을 추가한다: 역할(선택, A/B/팀장), 상태(선택, 준비/진행/검토/완료), 마감일(날짜), 문서 링크(URL), 검토 메모(텍스트).
  4. 각 창에서 복사한 Google Docs/Sheets/Slides 공유 URL을 해당 행의 문서 링크 속성에 붙여넣는다.
  5. 상단에 오늘의 목표 컬럼 또는 토글 블록을 추가해 작업 우선순위를 적고, 상태가 바뀔 때마다 즉시 업데이트한다.

성과물

프로젝트 보드 테이블이 완성된 Notion 페이지 스크린샷과 공유 링크를 준비한다.

<보완요청> Notion 테이블 구성 예시 이미지 삽입

A 담당자가 ChatGPT와 Google Docs로 전략 기획 아이디어를 생성하고 협업 피드백을 반영합니다.

실습 절차

  1. ChatGPT에 아래 프롬프트를 입력합니다.
"대학생 취업 전략 프로젝트를 위해 직무 역량·네트워킹·포트폴리오 강화를 각각 3단계 실행 계획으로 제안해줘.
각 단계에 필요 도구, 예상 산출물, KPI 후보를 포함해줘."
  1. 출력 결과를 Docs 템플릿에 붙여넣고 제안 모드로 전환합니다.
  2. B 담당자와 팀장은 댓글로 보완 의견을 남기고 해결된 항목은 "해결됨" 처리합니다.
  3. 버전 이력에 "브레인스토밍 확정본"을 저장한 뒤 링크를 Notion 프로젝트 보드 행에 첨부합니다.

성과물

Docs 최종 링크와 핵심 의사결정 요약 3줄을 Notion 회의록 섹션에 기록합니다.

<보완요청> Docs 제안 모드 전/후 비교 이미지 삽입

A 담당자가 KPI를 정리하고 Google Sheets에서 시각화 대시보드를 구축합니다.

데이터 형식 안내: CSV(Comma Separated Values)와 TSV(Tab Separated Values)는 표 데이터를 텍스트로 내보낼 때 사용하는 대표 형식입니다. TSV는 열 사이를 탭으로 구분하기 때문에 Google Sheets에 그대로 붙여넣기 용이하며, 수강생들이 가끔 "css"라고 부르는 서식 파일과는 다릅니다.

  1. ChatGPT에 아래 프롬프트를 입력해 TSV 표 초안을 생성합니다.
"대학생 취업 전략 프로젝트를 위해 직무 역량·네트워킹·포트폴리오 강화를 각각 3단계 실행 계획으로 제안해줘. 열은 목표	지표명	측정 방법	기준값	목표값	측정 주기	책임자로 구성하고 TSV(탭 구분) 형식으로 출력해줘."
  1. 시크릿 창 1(A)에서 Google Sheets 새 문서를 열고(A1 셀 선택) TSV 결과를 그대로 붙여넣습니다.
  2. 상단 메뉴 형식 → 조건부 서식을 열어 현재 진행률 열에 색상 눈금(빨강-노랑-초록)을 적용합니다. 범위는 자동 감지값(예: E2:E10)을 확인하고 필요 시 조정합니다.
  3. 상단 메뉴 데이터 → 필터 만들기로 필터를 추가한 뒤, 역할 또는 지표 유형별로 정렬 가능한 드롭다운을 확인합니다.
  4. 상단 메뉴 삽입 → 차트를 선택해 차트 편집기에서 차트 유형을 막대형으로 설정하고, 동일 편집기 설정 → 데이터 범위에서 전체 표(A1:G... )를 지정합니다. 사용자 지정 탭에서 색상을 브랜드 컬러로 변경한 뒤 연결 유지 옵션이 켜져 있는지 확인합니다.

실습 팁

  • TSV는 각 열이 탭으로 구분되므로 붙여넣기만으로 자동 분리됩니다. CSV(쉼표 구분)보다 빠르고 정렬이 깨지지 않습니다.
  • 조건부 서식 규칙을 추가할 때 형식 → 조건부 서식 → 색상 스케일을 선택하면 수치에 따라 색상이 자동 적용됩니다.
  • 차트는 삽입 → 차트 이후 오른쪽 패널에서 사용자 지정 → 범례로 이동해 글씨체·색상을 프로젝트 톤에 맞출 수 있습니다.

성과물

Sheets 공유 링크와 차트 URL을 Docs·Notion 프로젝트 보드 행에 첨부합니다.

<보완요청> 조건부서식 적용된 KPI 시트 이미지 삽입

A 담당자가 Google Slides 한 장으로 전략 스토리를 시각화합니다.

  1. Slides에서 삽입 → 차트 → 시트에서를 클릭하고 앞 단계에서 만든 KPI 시트를 선택합니다. 연결 유지 체크박스를 확인해 Sheets와 실시간 연동을 유지합니다.
  2. 슬라이드 레이아웃을 구분하기 위해 삽입 → 텍스트 상자로 상단에 전략 요약을 입력하고, 삽입 → 도형 → 직사각형을 이용해 오른쪽에는 KPI 그래프 영역, 하단에는 삽입 → 선으로 실행 타임라인을 구성합니다.
  3. ChatGPT에 "30초 엘리베이터 피치"를 요청해 복사한 뒤, Slides에서 보기 → 발표자 노트를 열어 해당 내용을 붙여넣습니다.
  4. 디자인을 통일하고 싶다면 슬라이드 → 테마 변경 또는 오른쪽 테마 패널에서 브랜드 색상을 선택합니다.
  5. 작업 후 파일 → 버전 기록 → 현재 버전을 이름 지정을 선택해 전략 요약 v1으로 저장하고, 오른쪽 상단 공유 버튼에서 링크 있는 모든 사용자를 댓글 가능으로 설정합니다.

성과물

Slides 공유 링크와 파일 → 다운로드 → PDF 문서(.pdf)로 내려받은 파일을 Notion 회고 섹션에 /파일 블록으로 업로드합니다.

<보완요청> Slides 완성본 썸네일 삽입

B 담당자가 프로젝트 경험 기반 자기소개서를 작성하고 팀 피드백을 반영합니다.

  1. 다음 프롬프트로 초안을 생성합니다.
"전략 기획 프로젝트에서 데이터 기반 의사결정을 이끈 경험을 500자 이내 자기소개서로 작성해줘.
도입-과제-행동-성과-다짐 구조를 따르고 KPI 수치를 포함해줘."
  1. Docs에 붙여넣은 뒤 제안 모드로 전환합니다.
  2. A 담당자와 팀장은 표현 명료성·정량 근거·협업 역할 관점에서 검토합니다.
  3. **파일 → 다운로드 → PDF 문서(.pdf)**를 선택해 로컬에 저장한 뒤, 오른쪽 상단 공유 버튼에서 링크 있는 모든 사용자를 댓글 가능으로 설정하고 링크를 복사합니다.

검토 포인트

  • [ ] 핵심 문장마다 구체적 수치가 포함되어 있는가?
  • [ ] 문장 길이가 20~25자 범위 안에서 읽기 쉬운가?
  • [ ] 협업 경험과 개인 기여가 명확히 구분되는가?

성과물

Docs 공유 링크와 PDF 파일을 Notion 회고 섹션 /파일 블록에 업로드하고, 수정 요약을 코멘트 필드에 기록합니다.

<보완요청> Docs 제안 모드 캡처 삽입

팀장이 모든 산출물을 Notion 프로젝트 페이지에 정확하게 연결하고 회고 로그를 정리합니다.

  1. 프로젝트 보드에서 각 마일스톤 행을 열고 문서 링크 속성에 Docs·Sheets·Slides 공유 URL을 붙여넣습니다. 필요 시 / 입력 후 북마크 블록을 추가해 시각적 미리보기도 남깁니다.
  2. 첨부 속성을 추가해 PDF, 이미지 등의 산출물을 /파일 블록으로 업로드하고, 상태 속성을 검토 또는 완료로 업데이트합니다.
  3. 페이지 하단에 /데이터베이스 - 테이블을 입력해 회고(Log) 데이터베이스를 만들고 구분(Keep/Problem/Try)·작성자·링크 속성을 추가합니다.
  4. /할 일 블록으로 5강 준비 체크리스트를 생성해 포트폴리오용 이미지, 텍스트, KPI 표 링크를 확인하며 체크합니다.

성과물

최신 상태의 Notion 공유 링크와 회고(Log) 데이터베이스 요약 3줄을 팀 채팅과 담당 교수에게 전달합니다.

<보완요청> Notion 회고 보드 캡처 삽입

팀 전체가 품질 점검 리스트를 확인합니다.

  • [ ] Docs·Sheets·Slides의 공유 버튼에서 링크 있는 모든 사용자 → 댓글 가능이 적용되었는지 점검한다.
  • [ ] ChatGPT 대화 URL을 Notion 참고 자료 구역에 /북마크 블록으로 모아 놓았는지 확인한다.
  • [ ] 프로젝트 보드의 상태 속성이 모두 완료 또는 검토로 업데이트되었는지 확인한다.
  • [ ] 회고(Log) 데이터베이스에 팀원별로 최소 1개 이상의 Keep/Problem/Try 기록이 있는지 확인한다.

검증 후 각 공유 링크를 브라우저 다른 창에서 열어 접근 권한이 정상인지 최종 점검합니다.
  • 직무별 맞춤 전략 요약을 ChatGPT로 생성해 Notion 자료실에 추가
  • Slides를 GIF 애니메이션으로 변환해 포트폴리오 삽입 가능성 테스트
  • 다른 팀과 산출물을 교차 피드백하는 패널 토론 주제 선정

확장 미션은 선택 사항이며, 수행 결과를 5강 준비 자료에 첨부하면 추가 피드백을 받을 수 있습니다.
